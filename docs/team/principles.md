# 产品原则

[返回索引](../INDEX.md)

## 核心价值观

### 1. 稳定优先

> 系统稳定性高于功能丰富度

- 消息不丢失、不重复
- 服务高可用（99.9%+）
- 异常自动恢复
- 数据持久可靠

### 2. 用户隐私

> 保护用户数据安全是底线

- Cookie 加密存储
- 敏感信息不落日志
- 数据隔离清晰
- 权限最小化原则

### 3. 简单易用

> 降低使用门槛，让非技术人员也能轻松上手

- Web 界面直观
- 配置简单明了
- 错误提示友好
- 文档完善

### 4. 可扩展性

> 支持业务增长，架构灵活演进

- 多账号支持
- 水平扩展能力
- 插件化设计
- 配置驱动

## 设计决策

### 为什么选择 SQLite？

| 考量 | 决策 |
|------|------|
| 部署简单 | ✅ 无需独立数据库服务 |
| 数据量 | ✅ 单用户场景足够 |
| 运维成本 | ✅ 零运维 |
| 扩展性 | ⚠️ 可迁移至 PostgreSQL |

### 为什么选择 FastAPI？

| 考量 | 决策 |
|------|------|
| 性能 | ✅ 异步高性能 |
| 开发效率 | ✅ 自动 API 文档 |
| 类型安全 | ✅ Pydantic 集成 |
| 生态 | ✅ 丰富的中间件 |

### 为什么选择 WebSocket？

| 考量 | 决策 |
|------|------|
| 实时性 | ✅ 毫秒级消息 |
| 效率 | ✅ 长连接复用 |
| 兼容性 | ✅ 闲鱼官方协议 |

## 用户体验原则

### 反馈及时

- 操作立即响应
- 加载状态明确
- 错误提示具体
- 成功确认清晰

### 操作简单

- 减少点击次数
- 默认值合理
- 批量操作支持
- 快捷键可用

### 信息清晰

- 层级分明
- 重点突出
- 数据可视化
- 帮助文档易找
