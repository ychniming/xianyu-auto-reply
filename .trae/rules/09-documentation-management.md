# 文档规则

## 核心原则

**Agent-First**: 文档是智能体的知识来源，必须准确、新鲜、可发现。

过多指导反而无效：当一切都"重要"时，一切都不重要了。

## 文档组织结构

```
docs/
├── INDEX.md              # 知识库总览（必须）
├── architecture/         # 架构文档
│   ├── overview.md      # 顶层地图
│   └── domains.md       # 领域划分
├── guides/              # 操作指南
├── plans/               # 执行计划（一等公民）
│   ├── active/         # 活跃计划
│   └── completed/      # 已完成计划
└── reference/          # 参考资料
```

## 文档类型

| 类型 | 描述 | 存储位置 |
|-----|------|---------|
| PRD | 产品需求文档 | docs/PRD.md |
| Architecture | 架构设计文档 | docs/architecture/ |
| Database | 数据库设计文档 | docs/Database.md |
| API | API接口文档 | docs/API.md |
| DesignTokens | 设计令牌文档 | docs/DesignTokens.md |

## AGENTS.md 规则

**限制：最多 100 行**

AGENTS.md 是知识库的目录/索引，而非百科全书。指向 docs/ 下的深度知识。

## 文档结构

每个文档必须包含：
```yaml
---
title: [标题]
description: [简短描述]
lastUpdated: YYYY-MM-DD
maintainer: [维护者]
---
```

## 版本管理

- 语义化版本
- 变更日志
- 更新日期（lastUpdated 字段）
- 文档存储在代码仓库中（版本控制）

## 验证规则

| 规则 ID | 规则名称 | 严重级别 |
|---------|---------|---------|
| DLV-001 | 必须包含时间戳 | Error |
| DLV-002 | 链接必须有效 | Error |
| DLV-003 | AGENTS.md ≤100 行 | Error |
| DLV-004 | 最小交叉链接数 | Warning |

## 新鲜度规则

| 评级 | 时间范围 | 处理方式 |
|------|---------|---------|
| A | 7天内更新 | 合格 |
| B | 30天内更新 | 合格 |
| C | 90天内更新 | 需要审查 |
| D | 超过90天 | 必须更新或删除 |

## 存储位置规则

| 知识类型 | 存储位置 | 理由 |
|---------|---------|------|
| 代码相关 | docs/ | 版本控制，智能体可访问 |
| 临时讨论 | 不存储 | 无持久价值 |
| 决策记录 | docs/plans/ | 可追溯 |
| 团队文化 | docs/team/ | 集中管理 |

## 维护策略

### 每日
- 扫描过期文档（>90天）
- 验证链接有效性

### 每周
- 全量质量评分更新
- 检查文档覆盖率
- 扫描孤立文档

### 自动化
- CI 检查文档新鲜度和链接
- doc-gardening agent 扫描过时文档并发起修复 PR

## 代码注释
- 使用中文注释
- 复杂逻辑添加说明
- 公共API添加文档字符串

## 更新规则
- 功能变更同步更新文档
- 架构调整记录决策原因
- 过时文档视为技术债务，及时偿还
